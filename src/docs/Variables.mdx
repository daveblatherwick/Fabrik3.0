# Variables

Every design decision in Fabric resolves to a CSS custom property under the `--fabric-*` prefix. Components consume these variables, so switching theme/brand/density at the root cascades to every rendered component automatically.

## Architecture

Fabric's Figma file defines **9 variable collections** and **941 variables**, organised into three layers:

1. **Primitives** — raw dimensions, hex values, font families. Not used directly by components.
2. **Semantic tokens** — aliased primitives with meaning (`text-primary`, `bg-brand-solid`, `border-error`). These are what components consume.
3. **Component tokens** — component-specific aliases (`button-brand-bg`, `input-primary-border`) built on semantic tokens.

The indirection lets brand, color mode, and density cascade independently without the component code knowing about them.
