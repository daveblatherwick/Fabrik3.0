# Docs template

Every component in this Storybook ships with a standard docs header. That's the rectangle at the top of [Button](../?path=/docs/components-button--docs) and [OrderBook](../?path=/docs/exchange-components-orderbook--docs) — status, Figma link, design notes, a11y, limitations, and a changelog.

This page is how you add it to a new component.

## How to use it

Two files per component. The `.stories.tsx` is unchanged — it defines stories. The new `.mdx` file next to it becomes the Docs tab.

```tsx
// Button.stories.tsx — unchanged
export default {
  title: "Components/Button",
  component: Button,
  tags: ["autodocs"],
  // ...
};
```

```mdx
// Button.mdx — replaces the auto-generated Docs page
import { Meta, Title, Primary, Controls, Stories } from "@storybook/addon-docs/blocks";
import { ComponentMeta } from "../../docs/ComponentMeta";
import * as ButtonStories from "./Button.stories";

<Meta of={ButtonStories} />

<Title />

One sentence describing what the component is for.

  status="alpha"
```

## Props reference

| Prop | Type | Notes |
| ---- | ---- | ----- |
| `status` | `"alpha" \| "beta" \| "stable" \| "deprecated"` | Colored badge — signals maturity to consumers. |
| `version` | `string` | Semver — bump on every meaningful change. |
| `owner` | `string` | Single person or team responsible. |
| `lastUpdated` | `string` (YYYY-MM-DD) | Usually matches the newest changelog entry. |
| `figma` | `FigmaRef \| FigmaRef[]` | URL + optional `label` + optional `nodeId`. Array when a component maps to multiple Figma frames. |
| `designNotes` | `ReactNode[]` | Non-obvious design decisions. Why, not what. |
| `a11yNotes` | `ReactNode[]` | Concrete a11y expectations this component meets. |
| `limitations` | `ReactNode[]` | Things that aren't implemented yet. State the gap plainly. |
| `changelog` | `ChangelogEntry[]` | **Newest first.** Date + optional version + bullet list of changes. |

## When to bump each thing

- **Status**: Promote `alpha → beta → stable` as the component hardens. Demote to `deprecated` when it has a replacement.
- **Version**: Patch for bug fixes and visual tweaks, minor for new props/variants, major for breaking API changes.
- **Changelog**: Every PR that touches the component. If the change isn't worth documenting, consider whether it needs to ship.
- **Limitations**: Update when you add the limited feature — delete the entry, don't cross it out.
